# Ads Relatório

Gere relatórios de performance de tráfego pago em múltiplos formatos: WhatsApp (resumo rápido), PDF (completo visual) e diagnóstico estratégico com IA.

## 1. Tipos de relatório

| Tipo | Frequência | Formato | Destinatário |
|---|---|---|---|
| Flash diário | Todo dia 18h | WhatsApp | Marco (interno) |
| Semanal | Segunda 9h | PDF + WhatsApp | Marco / equipe |
| Mensal | Dia 1 | PDF completo | Marco / cliente |
| Diagnóstico | Sob demanda | Texto estruturado | Estratégico |

## 2. Flash diário (WhatsApp)

### Formato
```
📊 ADS — [dia/mês]

GOOGLE:
💰 R$ XXX investidos
👆 XXX cliques (CPC R$ X.XX)
🎯 X conversões (CPA R$ XX)

META:
💰 R$ XXX investidos
👆 XXX cliques (CTR X.X%)
🎯 X conversões (CPA R$ XX)

TOTAL: R$ X.XXX → X conv (CPA R$ XX)

[▲/▼] vs ontem: [variação principal]
[⚠️ alerta se houver]
```

### Regras
- Max 500 caracteres (leitura em 10 segundos)
- Só métricas que mudaram significativamente
- Alerta apenas se CPA > meta ou conversão = 0
- Não enviar se dia sem gasto (final de semana sem campanha)

## 5. Diagnóstico estratégico (IA)

### Input
- Dados dos últimos 30 dias (ambos canais)
- Histórico de 3 meses (tendências)
- Dados de vendas reais (cruzamento)
- Objetivos de negócio (meta de leads, CAC máximo)

### Output
```
═══ DIAGNÓSTICO ESTRATÉGICO — [mês/ano] ═══

CONTEXTO:
[1-2 linhas sobre o momento do negócio e objetivos]

SAÚDE DAS CAMPANHAS: [Score X/10]

PONTOS FORTES:
1. [insight positivo com dados]
2. [insight positivo]

PROBLEMAS IDENTIFICADOS:
1. [problema + impacto financeiro estimado]
2. [problema + causa provável]

OPORTUNIDADES NÃO EXPLORADAS:
1. [oportunidade + potencial estimado]
   Como: [passo concreto]
2. [oportunidade]
   Como: [passo concreto]

PLANO DE AÇÃO (próximos 30 dias):
| Prioridade | Ação | Canal | Impacto | Esforço |
|---|---|---|---|---|
| 1 | [ação] | Google | Alto | Baixo |
| 2 | [ação] | Meta | Alto | Médio |
| 3 | [ação] | Ambos | Médio | Baixo |

PROJEÇÃO:
Se executar o plano: CPA pode cair X% → economia de R$ XXX/mês
Se manter atual: tendência de [X] nos próximos 30 dias

BENCHMARK:
- CPA setor (construção B2B): R$ XX-XX
- CTR médio setor: X.X%
- Posição Pousinox: [acima/na média/abaixo]
```

## 6. Geração automatizada

### Via Routine (recomendado)
```yaml
# Flash diário
trigger: schedule "0 18 * * 1-5"  # Seg-Sex 18h
action: coletar dados → gerar flash → enviar WhatsApp

# Semanal
trigger: schedule "0 9 * * 1"  # Segunda 9h
action: coletar 7d → gerar PDF → enviar WhatsApp+email

# Mensal
trigger: schedule "0 9 1 * *"  # Dia 1 às 9h
action: coletar 30d → gerar PDF completo → enviar
```

### Via skill (sob demanda)
```
Usuário: /ads-relatorio
→ Pergunta: Qual tipo? (flash/semanal/mensal/diagnóstico)
→ Pergunta: Período? (default: últimos 7 dias)
→ Gera e entrega no formato escolhido
```

## 7. Integração com pipeline Pousinox

```
/ads-dashboard (dados)
    ↓
/ads-otimizar (ações)
    ↓
/ads-relatorio (comunicação)
    ↓
/routines (automação 24/7)
```

Dados fluem:
- Google/Meta APIs → Dashboard → Relatório
- Supabase (leads, vendas) → Funil real → ROI
- IA (diagnóstico) → Plano de ação → Otimização

## 9. Quando usar
- Diário 18h: flash automático (routine)
- Segunda: relatório semanal
- Dia 1: relatório mensal completo
- Sob demanda: diagnóstico antes de decisão de budget
- Ao apresentar resultados para sócio/investidor
- Ao justificar aumento ou corte de investimento
