# Copy LP

Gere a estrutura completa de copy para landing pages antes de implementar o design. Texto persuasivo baseado em metodologias validadas (AIDA, PAS, 4Ps).

## 1. Briefing

Pergunte ao usuário:
- **Produto/Serviço:** O que está vendendo?
- **Público-alvo:** Quem compra? (cargo, segmento, dor)
- **Problema principal:** Qual dor resolve?
- **Diferencial:** Por que escolher você e não o concorrente?
- **Ação desejada:** O que o visitante deve fazer? (WhatsApp, formulário, comprar)
- **Tom:** Técnico / Emocional / Direto / Educativo?
- **Agressividade:** 1-10 (1=institucional suave, 10=copy agressiva de lançamento)

## 2. Frameworks de estrutura

### AIDA (padrão — funciona para 80% dos casos)
```
A — Atenção: Headline que para o scroll
I — Interesse: Contexto do problema + identificação
D — Desejo: Solução + benefícios + prova
A — Ação: CTA claro e urgente
```

### PAS (melhor para dor forte)
```
P — Problema: Agitar a dor (mostrar que entende)
A — Agitação: Consequências de não resolver
S — Solução: Apresentar a saída + CTA
```

### 4Ps (melhor para produtos técnicos/B2B)
```
P — Promessa: Resultado específico
P — Pintura: Cenário do resultado alcançado
P — Prova: Dados, laudos, cases
P — Push: Empurrão final (urgência/escassez)
```

## 3. Blocos da landing page

### Bloco 1 — Hero (Above the fold)
```
HEADLINE: [Max 10 palavras — promessa principal]
SUBHEADLINE: [1-2 linhas — como entrega a promessa]
CTA PRIMÁRIO: [Texto do botão — ação clara]
PROVA RÁPIDA: [1 linha — "Usado por X empresas" ou "Laudo SENAI"]
```

Regras:
- Headline responde: "O que ganho?" (não "o que é")
- Subheadline responde: "Como?" ou "Para quem?"
- CTA = verbo + benefício ("Solicitar orçamento sem compromisso")
- Sem jargão, sem "bem-vindo ao nosso site"

### Bloco 2 — Problema/Dor
```
TÍTULO DA SEÇÃO: [Pergunta que gera identificação]

DOR 1: [Situação frustrante que o público vive]
DOR 2: [Consequência de não resolver]
DOR 3: [Medo ou risco real]

TRANSIÇÃO: "E se existisse uma solução que..." / "Não precisa ser assim"
```

Regras:
- Usar linguagem do cliente (não técnica)
- Ser específico (não "problemas de qualidade" → "peças soltando da fachada em 2 anos")
- Max 3 dores (mais que isso dilui)

### Bloco 3 — Solução
```
TÍTULO: [Como resolve — em linguagem de benefício]

BENEFÍCIO 1: [Feature → Benefício → Resultado]
  "Aço inox 304 → Não corrói → Dura 30+ anos sem manutenção"

BENEFÍCIO 2: [Feature → Benefício → Resultado]
  "Projeto sob medida → Encaixa perfeito → Zero desperdício de material"

BENEFÍCIO 3: [Feature → Benefício → Resultado]
  "Laudo técnico SENAI → Segurança comprovada → Responsabilidade técnica garantida"
```

Regras:
- Nunca listar features sozinhas (sempre traduzir em benefício)
- Fórmula: "Isso significa que você..."
- Visual: ícone + título curto + 1 linha de explicação

### Bloco 4 — Prova social / Autoridade
```
TÍTULO: [Estatística ou afirmação de autoridade]

PROVAS:
- Número de clientes/projetos atendidos
- Laudo/certificação (SENAI, LAMAT, ISO)
- Cases com resultado mensurável
- Depoimentos (nome + empresa + resultado)
- Logos de clientes conhecidos
```

Regras:
- Números > palavras ("142 projetos entregues" > "muita experiência")
- Depoimento ideal: dor antes → solução → resultado depois
- Se não tem depoimento: usar dados técnicos como prova

### Bloco 5 — Como funciona (processo)
```
TÍTULO: "Como funciona" / "3 passos para [resultado]"

PASSO 1: [Ação simples] — "Envie suas medidas"
PASSO 2: [O que acontece] — "Projetamos a solução ideal"
PASSO 3: [Resultado] — "Receba instalado e com garantia"
```

Regras:
- Max 3-5 passos (simplicidade)
- Cada passo: 1 verbo + resultado
- Reduzir percepção de complexidade/risco

### Bloco 6 — Objeções (FAQ ou comparativo)
```
TÍTULO: "Dúvidas frequentes" / "[Produto] vs alternativas"

OBJEÇÃO 1: "É caro?" → [Reframe: custo vs investimento de longo prazo]
OBJEÇÃO 2: "Demora?" → [Prazo real + comparativo]
OBJEÇÃO 3: "Funciona para meu caso?" → [Versatilidade + cases variados]
OBJEÇÃO 4: "E se der problema?" → [Garantia + suporte]
```

Regras:
- Antecipar as 3-5 objeções mais comuns
- Resposta curta (2-3 linhas max)
- Tom: seguro, não defensivo
- FAQ ajuda SEO (schema markup)

### Bloco 7 — Oferta / Preço (se aplicável)
```
TÍTULO: "Investimento" / "Escolha seu plano"

ÂNCORA DE PREÇO: [Valor percebido maior que o preço]
PREÇO: [Valor claro, sem surpresas]
CONDIÇÕES: [Pagamento, parcelamento]
BÔNUS: [O que ganha a mais — laudo grátis, frete, suporte]
GARANTIA: [Reduzir risco — "30 dias ou devolvemos"]
```

### Bloco 8 — CTA Final
```
HEADLINE DE FECHAMENTO: [Reforço da promessa principal]
SUBTEXT: [Urgência ou escassez real]
CTA: [Mesmo botão do hero — consistência]
MICRO-COPY: [Abaixo do botão: "Sem compromisso" / "Resposta em até 2h"]
```

Regras:
- Repetir CTA pelo menos 3x na página (hero, meio, final)
- Urgência real (não falsa): "Fila de produção: entrega em X dias"
- Micro-copy reduz fricção

## 4. Gatilhos mentais

| Gatilho | Onde usar | Exemplo Pousinox |
|---|---|---|
| Autoridade | Prova social | "Laudo técnico LAMAT/SENAI" |
| Prova social | Cases, números | "142 projetos em 8 estados" |
| Escassez | CTA, oferta | "Fila de produção: próxima vaga em X dias" |
| Reciprocidade | Lead magnet | "Calculadora grátis de materiais" |
| Especificidade | Números | "Resiste a 1.200 kgf de tração" |
| Antecipação | Processo | "Em 48h você recebe o projeto técnico" |
| Segurança | Garantia | "Garantia de 10 anos contra corrosão" |
| Pertencimento | Prova social | "Construtoras como X e Y já utilizam" |

## 5. Headlines — Fórmulas

### Fórmulas testadas
```
1. [Resultado desejado] sem [objeção principal]
   "Fachada de porcelanato sem risco de desplacamento"

2. Como [público] [consegue resultado] em [prazo/condição]
   "Como construtoras eliminam retrabalho em fachadas"

3. [Número] [coisas] que [consequência]
   "3 erros na fixação que comprometem a garantia da obra"

4. A verdade sobre [tema controverso]
   "A verdade sobre fixadores galvanizados em fachada"

5. [Resultado] + [prova]
   "30 anos sem manutenção — comprovado em ensaio técnico"

6. Pare de [dor] — [solução]
   "Pare de perder dinheiro com retrabalho em fachadas"
```

### Headlines Pousinox (banco)
```
- "Fixação de porcelanato com garantia de engenharia"
- "Aço inox + laudo técnico = zero risco na sua obra"
- "O fixador que construtoras especificam por segurança"
- "De porcelanato na fachada a cliente satisfeito — sem surpresas"
- "Projetos sob medida direto da fábrica — sem intermediário"
- "Fixador que aguenta: 1.200 kgf testados em laboratório"
```

## 6. CTAs — Banco de opções

### Por nível de compromisso (menor → maior)
```
Baixo: "Ver catálogo" / "Baixar ficha técnica" / "Calcular materiais"
Médio: "Solicitar orçamento" / "Falar com especialista" / "Agendar visita técnica"
Alto:  "Comprar agora" / "Fechar pedido" / "Contratar projeto"
```

### Por canal
```
WhatsApp: "Falar pelo WhatsApp" / "Tirar dúvidas agora" / "Pedir orçamento no WhatsApp"
Formulário: "Solicitar proposta" / "Receber contato" / "Enviar medidas"
Telefone: "Ligar agora" / "Falar com engenheiro"
```

### Micro-copy (abaixo do botão)
```
"Sem compromisso — cancele quando quiser"
"Resposta em até 2 horas úteis"
"Mais de 100 empresas já solicitaram"
"Orçamento gratuito e sem pressão"
```

## 7. Tom de voz por agressividade

| Nível | Tom | Quando usar |
|---|---|---|
| 1-3 | Institucional, informativo | Site principal, B2B enterprise |
| 4-6 | Persuasivo, benefício-foco | Landing pages de produto |
| 7-8 | Direto, urgente, emocional | Campanhas de ads, promoção |
| 9-10 | Agressivo, escassez, FOMO | Lançamentos, Black Friday |

### Pousinox padrão: nível 5-6
- Profissional mas não frio
- Focado em benefício e prova técnica
- Sem exageros ("melhor do mundo") — usa dados reais
- Urgência natural ("fila de produção") não artificial
