# Briefing

Transforme uma ideia vaga em um briefing completo e executável — para design, desenvolvimento, campanha ou projeto.

## 1. Capturar a ideia
Pergunte ao usuário:
- Qual a ideia/necessidade? (em suas palavras)
- Tipo: Design / Desenvolvimento / Campanha / Projeto / Conteúdo / Produto?
- Para quem? (público/equipe que vai executar)
- Prazo desejado?
- Budget (se aplicável)?
- Referências ou inspirações?

## 2. Estruturar por tipo

### Briefing de Design
- **Objetivo:** O que precisa ser criado
- **Onde será usado:** Digital (tamanhos), impresso (formato)
- **Público:** Quem vai ver
- **Mensagem principal:** O que deve comunicar
- **Tom visual:** Moderno/clássico, minimalista/rico, cores
- **Elementos obrigatórios:** Logo, textos, CTAs, fotos
- **Referências:** Links ou descrições do que gosta
- **Entregáveis:** Formatos e quantidade de peças
- **Prazo:** Data de entrega

### Briefing de Desenvolvimento
- **Objetivo:** Feature/módulo/correção
- **Contexto:** Por que é necessário (problema que resolve)
- **Escopo:** O que está incluído e o que NÃO está
- **Requisitos funcionais:** Lista numerada do que deve fazer
- **Requisitos técnicos:** Stack, integrações, performance
- **UI/UX:** Wireframe textual ou referência visual
- **Critérios de aceite:** Como saber que está pronto
- **Prioridade:** P0 (crítico) / P1 (importante) / P2 (nice-to-have)
- **Estimativa:** Complexidade (P/M/G)

### Briefing de Campanha
- **Objetivo:** Awareness / Leads / Vendas / Reativação
- **Público-alvo:** Segmento, dor, estágio
- **Canais:** Meta / Google / Email / WhatsApp / Orgânico
- **Mensagem central:** Proposta de valor em 1 frase
- **Ofertas/CTAs:** O que oferecer, para onde direcionar
- **Budget:** Valor total e distribuição por canal
- **KPIs:** Métricas de sucesso (CPL, ROAS, leads)
- **Cronograma:** Fases (preparação, lançamento, otimização)
- **Materiais necessários:** Criativos, LPs, emails, copy

### Briefing de Conteúdo
- **Tema:** Assunto principal
- **Formato:** Blog / Post social / Vídeo / Email / Guia
- **Objetivo:** Educar / Vender / Engajar / SEO
- **Keywords:** Termos-alvo (se SEO)
- **Tom:** Técnico / Educativo / Inspiracional / Direto
- **Extensão:** Curto (300-500) / Médio (800-1200) / Longo (1500+)
- **CTAs:** Para onde direcionar o leitor
- **Referências:** Conteúdos similares de referência

## 3. Enriquecer automaticamente
Com base no contexto Pousinox:
- Adicionar dados de mercado relevantes (market_keywords)
- Sugerir segmentos prioritários
- Referenciar materiais existentes (laudos, cases)
- Alinhar com campanhas/projetos em andamento

## 4. Formato de entrega

```
★ BRIEFING — [Tipo] — [Título curto]
Data: [dd/mm/yyyy]
Solicitante: [nome]
Prioridade: [P0/P1/P2]

═══ CONTEXTO ═══
[Por que estamos fazendo isso — 2-3 linhas]

═══ OBJETIVO ═══
[O que queremos alcançar — 1 frase clara]

═══ ESCOPO ═══
Inclui:
- ...
- ...

NÃO inclui:
- ...

═══ DETALHAMENTO ═══
[Seções específicas por tipo — ver acima]

═══ ENTREGÁVEIS ═══
| # | Item | Formato | Prazo |
|---|---|---|---|
| 1 | ... | ... | ... |

═══ CRITÉRIOS DE SUCESSO ═══
- [ ] [Como saber que deu certo]
- [ ] [Métrica ou validação]

═══ REFERÊNCIAS ═══
- [link ou descrição]

═══ PRÓXIMOS PASSOS ═══
1. [Quem faz o quê primeiro]
2. [...]
```
